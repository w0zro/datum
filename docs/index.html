datum

One palette for your whole terminal — reverse-engineered from color science, where saturation encodes how often a token appears.

Get started Read the ideas GitHub ↗

the idea

Vivid, but hierarchical.

Every token carries a hue. But the loudness of that hue is inverse to how often the token appears — so the screen is colorful without becoming noise.

tier 1 · full chroma

The sparse reference points — strings, numbers, constants, types, function definitions — run at full saturation.

tier 2 · pastel glue

The ubiquitous ~75% — variables, calls, parameters, operators — take quiet pastels of the same hue, split from tier 1 by lightness (the one channel no color blindness collapses).

neutral

Pure punctuation and comments stay gray — earning less than plain text, so the color around them reads louder.

palette

Eleven colors, two tiers, both modes.

Anchored on the Okabe–Ito colorblind-safe hues, placed in OKLCH. Click any swatch to copy it — the WCAG / APCA numbers are computed live in your browser, so you can check the claims yourself.

ports

Theme the whole stack.

Every port is generated from the same palette, so they can't drift apart. Terminals and editors ship light + dark; shell tools follow your terminal automatically.

get started

One script, locally.

Clone the repo and run ./start. It builds every port and installs the ones you ask for — printing the exact line to enable each, never editing your configs behind your back.

# clone and see what's available
git clone https://github.com/w0zro/datum && cd datum
./start

# preview the palette + docs locally
./start docs

# apply to a terminal (or print how to enable it)
./start apply ghostty
./start apply all

# Neovim, via any plugin manager
Plug 'w0zro/datum'   "  then :colorscheme datum
